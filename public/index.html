Welcome to Algowhiz!

The easiest way to visualise searching & sorting algorithms such as

The A* Algorithm, a best-first search, meaning that it is formulated in terms of weighted graphs: starting from a specific starting node of a graph, it aims to find a path to the given goal node having the smallest cost (least distance travelled, shortest time, etc.). It does this by maintaining a tree of paths originating at the start node and extending those paths one edge at a time until the goal node is reached.

Dijkstra's Algorithm, a min-priority queue data structure for selecting the shortest paths known so far. Dijkstra's algorithm finds the shortest path from a given source node to every other node. It can also be used to find the shortest path to a specific destination node, by terminating the algorithm once the shortest path to the destination node is known.

And many more!

How does it work?

At the very heart of algowhiz lies an Express Node.js server, which parses the user sent C++ code (or the template A* Algorithm) using G++, a GNU compiler which is a part of the Windows Subsystem for Linux. The result of this code is then sent to the frontend JavaScript which carries out the visualisation:
